Project Plan
1. Strategy
The first section to building a site, is to identify the users needs and the business objectives. Making sure the content is relevant, Is the content culturally appropriate? Are the technologies being used relevant for your demographic? Is the business constantly improving and will information have to updated a regular basis? What's worth doing? What are we creating? What value does it provide? and much time do we have?
2. Scope
Then we can take a look at the functional specifications, what features will your site need to help your users? What content will be required to provide value, and what is the demographic that your trying to target as this will effect what content will be needed and how to display it.
3. Structure
Interaction design is important so the user can navigate themselves throughout your site. The order the navigation is displayed and also how many times the user must select an option before gaining access to their required information can rapidly change the users overall experience.
I'll then look at the Information architecture and will identify the relationships between the different information and group these together ready for the next steps.
4. Skeleton
Now that fitting information has been assembled together, it's important to recognise the hierarchy of the content, what will the user be subjected to first? This is also the case for the navigation making sure the content most relevant to the business goals get shown to the user on the initial load.
The design should be intuitive and also should compile with the standard web conventions such as the navigation location and the best arrangement and visual presentation of the elements.
Although this will likely be the first time your customer will be interacting with your site, they should be able to effortlessly locate their desired information and be able to complete tasks with little to no effort.
At this stage a wireframe or prototype of the site will be created to display the content and get the first visual look at your site. The benefits of this is that it can be changed and perfected quickly and easily, although is only a rough plan. This is still important so that we can make sure before the site goes live we already know what works best for the user and the company.
5. Surface
Finally, I can now work on developing the site and working on the colours, typography interface design reinforce the meaning of the content guide the user through task and information minimise cognitive overload and maximise intuitive learning Appropriate the content easily disguisable manner If content is KING, context is GOD. "Gary Vaynerchuck"

3. Request

Getting data from a form when it's sent to the route.

@app.route('/insert_name', methods=["POST"])
def insert_name():
    data = request.form['data']

4. Getting data

This will get all the data from a mongoDB collection.

@app.route('/insert_name', methods=["POST"])
def insert_name():
    data = mongo.db.collection.find()

4. Sorting data

This will sort the data coming from a mongoDB collection.

@app.route('/insert_name', methods=["POST"])
def insert_name():
    data = mongo.db.collection.find().sort([
    ('date', pymongo.ASCENDING)
    ])

4. Filtering data

This will filter data coming from a mongoDB collection.

@app.route('/insert_name', methods=["POST"])
def insert_name():
    data = mongo.db.collection.find({'name': 'alex'})

5. Updating data

Updates a chosen piece of data from mongoDB collections.

@app.route('/insert_name/<data_id>', methods=["POST"])
def insert_name(data_id):
    data = mongo.db.collection.remove({'_id': ObjectId(data_id)})

6. Removing data

This will allow us to remove data from a collection.

@app.route('/insert_name/<data_id>', methods=["POST"])
def insert_name(data_id):
    data = mongo.db.collection.remove({'_id': ObjectId(data_id)})

Heroku

Heroku is used to host projects that require a server, just as projects built using a backend framework. Heroku uses a ephemeral file system and everytime the worker restarts, approximately in 12 hours, all user uploaded content is deleted. To get around this, store media files on a external sorce such as Amazon Web Services.
